# คดีที่ 2 — "โน้ตหลุดออกไปแล้ว"

> Capo หน้าเครียดกว่าเดิม:
> "คราวนี้หนักกว่าเดิม... โน้ตอันตรายมันไม่ได้อยู่แค่ในบ้าน
> มัน **push ขึ้นไปบน remote ที่ทีมอื่นใช้ร่วม** เรียบร้อยแล้ว
> ของออกไปแล้ว เราลบประวัติเล่น ๆ ไม่ได้ เดี๋ยวพังกันทั้งองค์กร
> ไปจัดการให้เนียน — กลบมัน แต่ห้ามทำให้ของทีมพัง"

## สถานการณ์
- commit ล่าสุดมีไฟล์ `note.txt` (ข้อความที่ไม่ควรมี)
- commit นี้ **ถูก push ขึ้น `origin/main` ไปแล้ว**

## 🧠 ตอบก่อนลงมือ (คุยกับคู่)
1. ของชุดนี้หลุดออกไปแล้วหรือยัง? ต่างจากคดี 1 ตรงไหน?
2. ถ้าลบประวัติทิ้งบนของที่คนอื่นดึงไปแล้ว จะเกิดอะไรขึ้นกับทีม?
3. มีวิธีไหนที่ "ยกเลิกผลของ commit" โดยที่ **ไม่ลบ** commit เดิม?
4. แก้เสร็จในเครื่องแล้ว ต้องทำอะไรต่อให้ทีมเห็นด้วย?

## เป้าหมายให้ผ่าน
- `note.txt` หายจากของในมือ
- commit เดิมยัง **อยู่ในประวัติ** (ไม่ถูกลบ)
- remote อัปเดตแล้ว และประวัติบน remote ไม่ถูกล้าง
