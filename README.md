# THE FIXER 🧹

> เมืองนี้มีคนทำพลาดทุกวัน — และมีคนเดียวที่เก็บกวาดให้เรียบร้อยเสมอ
> เขาไม่ถาม เขาไม่บ่น เขาแค่ "จัดการ" ให้มันหายไป
> เขาคือ **The Fixer**

ลูกน้องในองค์กรทำพลาดไว้หลายอย่าง บางอันยังอยู่ในบ้าน บางอันหลุดออกไปข้างนอกแล้ว
หน้าที่ของคุณคือเข้าไป "เก็บงาน" ให้สะอาด ด้วยเครื่องมือที่ The Fixer ทุกคนต้องใช้เป็น — **Git**

แต่ละคดี **ไม่บอกว่าต้องใช้คำสั่งอะไร** — คุณต้องวินิจฉัยเองว่า "งานหลุดออกนอกบ้านหรือยัง"
แล้วเลือกวิธีให้ถูกกับสถานการณ์ นี่แหละที่แยก The Fixer ตัวจริง ออกจากคนที่แค่ทำตามสูตร

---

## กติกา
ทำงานเป็นคู่ — คนหนึ่งคุมคีย์บอร์ด อีกคนคิดและตรวจ แล้วสลับกันในแต่ละคดี

แต่ละคดีอยู่ในโฟลเดอร์ `case-X/`

| ไฟล์          | หน้าที่ |
| ------------- | ------- |
| `briefing.md` | โจทย์ + คำถาม(อ่านก่อนเสมอ) |
| `setup.sh`    | สร้าง "ที่เกิดเหตุ" ขึ้นมาในโฟลเดอร์ `scene/` |
| `verify.sh`   | ตรวจว่าเก็บงานสะอาดและ **ใช้วิธีถูก** หรือยัง |
| `hint.sh`     | เรียก Capo ขอคำใบ้ |

ที่ root repo มี `play.sh` — **คืนค่าโรงงาน** ล้างที่เกิดเหตุทุกคดีให้กลับสภาพเพิ่ง clone (ดูที่ข้างล่าง)

---

## ขั้นตอนการเล่น
```bash
เปิด terminal ให้พิมพ์ pwd เพื่อให้แน่ใจว่าอยู่ที่ .../the-fixer <<<=====

# 1) อ่านแฟ้มคดี — เปิด case-1/briefing.md กด Preview ขวาบน
#    (หรืออ่านใน terminal:  cat case-1/briefing.md)

# 2) สร้างที่เกิดเหตุ
bash case-1/setup.sh

# 3) เข้าไปลงมือเก็บงาน (เก็บกวาดให้สะอาดใช้ git ซะ)
cd case-1/scene # เข้ามาแล้ว
#   ... ลงมือ ...
cd ../.. # ออกมานอกสุดที่ root

# 4) ตรวจผลงาน
bash case-1/verify.sh
```

- **ผ่าน** → verify จะบอกคดีถัดไปที่ต้องรัน ไปต่อได้เลย
- **ไม่ผ่าน** → verify จะบอกว่าพลาดตรงไหนและให้คิดต่อ แก้แล้วรันใหม่ได้เรื่อย ๆ ไม่ต้องรอผู้สอน
- **ติดคิดไม่ออก** → `bash case-X/hint.sh` เรียก Capo ขอคำใบ้ (ใบ้วิธีคิด ไม่บอกคำสั่ง)
- **ถ้าพังคดีเดียว** → `bash case-X/setup.sh` ใหม่ ก็ได้ที่เกิดเหตุสด ๆ กลับมา

> ⚠️ verify ไม่ได้เช็คแค่ "ไฟล์หาย" แต่เช็ค **วิธี** ด้วย —
> ถ้าเลือกเครื่องมือผิดกับสถานการณ์ (เช่น ลบประวัติงานที่หลุดไปแล้ว) จะไม่ผ่าน

## คดีทั้งหมด
| คดี | ชื่อ | สิ่งที่ต้องตัดสินใจ |
| --- | ---- | ------------------ |
| 1 | ใบเสร็จยังไม่ออกจากบ้าน | ของยังไม่หลุด — เก็บยังไงให้ไม่เหลือร่องรอย? |
| 2 | โน้ตหลุดออกไปแล้ว | ของหลุดขึ้น remote ทีมแล้ว — กลบยังไงไม่ให้ทีมพัง? |
| 3 | เก็บเฉพาะของเสีย | ของเสียมีงานดีทับอยู่ — เล็งเก็บเฉพาะอันผิดได้ไหม? |

> ทำคดี 1 ให้ผ่านก่อนค่อยขยับ — เรียงจากง่ายไปยากแล้ว

## คืนค่าโรงงาน (เริ่มใหม่หมด)
อยากล้างทุกอย่างให้ repo กลับสภาพเพิ่ง clone — ลบที่เกิดเหตุทุกคดีทิ้ง:
```bash
bash play.sh
```
ลบเฉพาะของที่ generate ใหม่ได้ (`scene/`, remote จำลอง, คำใบ้) — **ไม่แตะไฟล์คดี**
หลังรันเสร็จ เริ่มเล่นใหม่ได้จาก `bash case-1/setup.sh` ตามปกติ

โชคดี Fixer 🤝
