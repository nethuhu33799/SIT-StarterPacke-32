# คดีที่ 3 — "เก็บเฉพาะของเสีย" (คดีปิดเกม)

> Capo จุดบุหรี่ มองเธอนิ่ง ๆ:
> "คดีสุดท้าย ดูฝีมือเธอ... ไฟล์ blacklist มันหลุดขึ้น remote ไปแล้ว
> แต่ปัญหาคือ — หลังจากนั้นมีงานดีของทีมตามขึ้นไปอีกหลายอย่าง
> เธอจะกลบของเสียโดยที่งานดีไม่หายได้ไหม? นั่นแหละที่แยก The Fixer ตัวจริงออกจากมือใหม่"

## สถานการณ์
- ประวัติมีลำดับ: ... → **commit ที่เพิ่ม `blacklist.txt` (ผิด)** → เพิ่ม `menu.txt` (ดี) → อัปเดตเวลาเปิดร้าน (ดี)
- ทุก commit **push ขึ้น remote แล้ว**
- commit ที่ผิด **ไม่ได้อยู่บนสุด** — มีงานดีทับอยู่

## 🧠 ตอบก่อนลงมือ (คุยกับคู่)
1. ถ้าย้อนทั้งสายกลับไปก่อน blacklist งานดีจะเป็นยังไง?
2. เราจะ "เล็ง" ยกเลิกเฉพาะ commit เดียวที่อยู่ตรงกลางได้ไหม? ด้วยอะไร?
3. หา commit hash ของอันที่ผิดยังไง?

## เป้าหมายให้ผ่าน
- `blacklist.txt` หาย — แต่ `menu.txt` และเวลาเปิดร้าน **ยังอยู่ครบ**
- commit เดิมยังอยู่ในประวัติ และ remote อัปเดต/ไม่ถูกล้าง
